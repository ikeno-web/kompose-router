# Requirements — kompose-router

## 1. ユーザーストーリー

### MUST（MVP）

| ID | 誰が | 何を | なぜ |
|----|------|------|------|
| US-01 | KMPアプリ開発者 | ルートを sealed class/interface で定義して画面遷移に使う | コンパイル時に無効なルートを検出し、文字列ベースのルーティングによる実行時クラッシュを排除するため |
| US-02 | KMPアプリ開発者 | 型安全な引数付きで画面遷移する（`navigate(Detail(id = "abc"))`） | 引数の型不一致や欠落をコンパイル時に防ぎ、ルーティングバグのデバッグ時間を削減するため |
| US-03 | KMPアプリ開発者 | バックスタックを操作する（push / pop / popToRoute / replace） | 一般的なナビゲーションパターン（戻る・巻き戻し・差し替え）をライブラリ側で安全に管理し、自前実装のバグを防ぐため |
| US-04 | KMPアプリ開発者 | 画面遷移にアニメーション（フェード・スライド・カスタム）を適用する | ユーザー体験を向上させつつ、Compose Animation API の直接利用に伴うボイラープレートを排除するため |
| US-05 | KMPアプリ開発者 | URI文字列からルートを解決してディープリンクを処理する | 外部URL（通知・ウェブリンク）からアプリ内の正しい画面へ遷移させるため |
| US-08 | KMPアプリ開発者 | ViewModel や非 @Composable コードからナビゲーションを実行する | ビジネスロジック層からの画面遷移（ログイン成功後の遷移等）を実現するため |
| US-10 | KMPアプリ開発者 | 現在のルートを StateFlow でリアクティブに購読する | ボトムナビゲーションのハイライト切替やアナリティクス送信など、ルート変更に反応する処理を宣言的に記述するため |

### SHOULD

| ID | 誰が | 何を | なぜ |
|----|------|------|------|
| US-06 | KMPアプリ開発者 | プロセス終了をまたいでナビゲーション状態を保存・復元する | Android のプロセス再生成やデスクトップアプリの再起動時にユーザーの画面位置を維持するため |
| US-07 | KMPアプリ開発者 | ネストしたナビゲーショングラフ（タブ・ボトムナビ）を作成する | 複雑な画面構造でタブごとに独立したバックスタックを管理するため |

### COULD

| ID | 誰が | 何を | なぜ |
|----|------|------|------|
| US-09 | KMPアプリ開発者 | ナビゲーションをインターセプトしてガード（認証チェック等）をかける | 未ログイン時のリダイレクトや権限チェックをナビゲーション層で一元管理するため |

### WON'T（v1.0 スコープ外）

| ID | 内容 | 理由 |
|----|------|------|
| W-01 | KSP/KAPT コード生成 | v1.0 はシンプルさ優先。sealed class + reified generics で十分な型安全性を提供 |
| W-02 | Android Lifecycle 統合 | Compose MP 共通コードに Android 依存を持ち込まない。消費者側で対応 |
| W-03 | マルチモジュール対応（別モジュールの Route 参照） | sealed interface の制約上、単一モジュール内でのルート定義を前提 |
| W-04 | アノテーションベースのディープリンク定義 | D-006（KSPなし）と矛盾。手動登録の `DeepLinkResolver` で対応 |

## 2. 非機能要件

| ID | 項目 | 目標値 | 計測方法 |
|----|------|--------|----------|
| NFR-01 | ナビゲーション遅延 | <16ms（60fps 1フレーム以内） | JVM ベンチマークで navigate() の実行時間を計測 |
| NFR-02 | バックスタックメモリ | 100エントリで <1MB | JVM ヒープダンプで BackStackEntry のリテイン量を計測 |
| NFR-03 | テストカバレッジ | 90%+ ラインカバレッジ | kover で計測 |
| NFR-04 | APIドキュメント | 公開API 100% KDoc | dokka で未文書警告ゼロ |

## 3. 用語集

| 用語 | 定義 |
|------|------|
| Route | 画面を表す sealed interface のサブタイプ。引数はデータクラスのプロパティとして型安全に保持 |
| Navigator | ルート間のナビゲーションを管理するクラス。@Composable 外からも使用可能 |
| BackStackEntry | バックスタックの1エントリ。一意ID + Route + 保存状態を持つ |
| RouteTransition | 画面遷移アニメーションの種類（Fade / SlideHorizontal / SlideVertical / Custom） |
| DeepLinkResolver | URIパターン → Route のマッピングを管理するリゾルバ |
| RouteGraphBuilder | Router DSL のビルダー。`screen<R> {}` で画面を登録する |

## 4. セルフチェック

- [x] 全ストーリーが「誰が・何を・なぜ」を含む
- [x] MUST が MVP として閉じている（ルート定義・遷移・バックスタック・アニメーション・ディープリンク・外部ナビゲーション・リアクティブ購読）
- [x] NFR が定量的に記載されている
- [x] WON'T でスコープ外を明示（KSP / Lifecycle / マルチモジュール / アノテーション）
- [x] D-006（KSPなし）との矛盾を排除（アノテーションベースDeepLinkを W-04 で除外）
- [x] US-01→MUST, US-02→MUST, ..., US-09→COULD のマッピング完了
